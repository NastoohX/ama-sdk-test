Quick Start Guide for Amazon EC2 VT2¶

  1. Launch an Amazon EC2 VT2 Instance

    • Follow the instructions at Launch an Amazon EC2 Instance

    • When choosing the instance type in the EC2 console, make sure to select the Amazon EC2 VT2 instance family.

    • When choosing an Amazon Machine Image (AMI), make sure to select version 3.0 of one of the Xilinx Video SDK AMI for VT1 Instances

    • To get more information about Amazon EC2 VT2 instances sizes and pricing see the VT1 web page

  2. Connect to the Amazon EC2 VT2 Instance

    • After launching the instance, follow the instructions in Connect to your instance to connect to the instance.

  3. Install the AMD AMA Video SDK examples by following the steps described in the package feed setup page.

  4. Set up the runtime environment:

    source startup.sh

    • Notes to be added

    • You are now ready to use the video acceleration features of your Amazon EC2 VT2 instance.

  5. Run your first examples

    • See the tutorials and examples page to learn how to run video transcoding jobs on your Amazon EC2 VT2 instance.


Installing the AMD AMA Video SDK on an existing AMI¶

In order to use a custom AMI on a Amazon EC2 VT2 instance, it is necessary to install the AMD AMA Video SDK in the AMI. This section explains how to do this. These steps are not necessary when using the public AMD AMA Video SDK AMI since it already includes a complete installation of the AMD AMA Video SDK and can readily be used on a Amazon EC2 VT2 instance.

Additionally, AWS provides packer scripts to make it easy for customers to build their own AMI’s for use with Amazon EC2 VT2 instances. Those scripts can be found at the following repo: https://github.com/aws-samples/aws-vt-baseami-pipeline

  • Ubuntu

    1. Download all relevant deb files in <PATH_TO_DEB_FILES> folder

    2. Install packages

      sudo apt -y install libhugetlbfs0 libboost-all-dev
dpkg --get-selections '*ma35*' | awk '{system("sudo apt -y purge " $1)}'
cd PATH_TO_DEB_FILES
sudo bash -c "apt -y install `pwd`/xilinx-ma35* `pwd`/ma35*"

      Ignore error message stating N: Download is performed unsandboxed as root as file...

  1. Perform a warm reboot of the instance.
